Core rules

Protocol baseline interfaces are mandatory. A provider that declares pi-provider-agent-v0 compatibility must implement every interface in this document.

Capabilities are not endpoint toggles. Provider profile and installed skills describe what work the agent can perform.

pi-works does not manage provider internals. Agent files and credential/model/skill configuration belong to the pi-os Owner Plane.

Authentication

Authorization: Bearer <provider-access-token>

Provider access tokens authorize pi-works to call Work Plane APIs. They are distinct from project work credentials installed by the agent owner.

Runs

Runs are assigned units of work.

POST /runs
GET /runs/:runId
POST /runs/:runId/cancel

Cancellation is a request. Providers should return resulting run state or a standard error.

Sessions

Sessions are the canonical work record. Reads are status queries and must not call AI.

GET /sessions?limit=25&cursor=<cursor>
GET /sessions/:sessionId

Session messages

GET /sessions/:sessionId/messages?limit=100&cursor=<cursor>

Messages are transcript records and are distinct from lower-level events.

Session events

GET /sessions/:sessionId/events?afterSeq=42&limit=100

Events are ordered records of state changes and progress. afterSeq returns events with seq > afterSeq.

Conversation

GET /conversations/:sessionId
POST /conversations/:sessionId/messages
POST /conversations/:sessionId/stop

Read is side-effect free. Send may create/queue a turn and trigger AI/agent work.

Session SSE event stream

GET /sessions/:sessionId/events/stream?afterSeq=42
Accept: text/event-stream

The SSE stream is one-way provider-to-Control-Center. Resume uses afterSeq or Last-Event-ID.

event: session.event
id: 43
data: {"seq":43,"type":"turn.started","payload":{"turnId":"turn_123"}}

Explicitly not Work Plane baseline

  • Agent files editing
  • Credential installation
  • Model config
  • Skills/extensions management
  • Provider filesystem allowlist
  • Owner audit
  • Provider runtime policy changes

pi-works does not expose Agent files UI; owner-plane file management belongs in pi-os owner surfaces.
